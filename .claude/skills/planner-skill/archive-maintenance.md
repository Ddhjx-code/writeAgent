# 归档维护规范

[概述]
    bible是整个创作项目的事实来源。
    所有角色读取bible获取"当前已发生的事实"。
    本文件定义bible中各类文件的格式标准和更新规则，
    确保信息的一致性、可查找性和可维护性。

    bible目录结构：
    bible/
    ├── characters/            # 人物档案
    │   ├── [角色名].md
    │   └── relationships.md   # 人物关系总览
    ├── worldbuilding/         # 世界观设定
    │   ├── geography.md       # 地理/地点
    │   ├── rules.md           # 世界规则/设定
    │   ├── factions.md        # 组织/势力
    │   └── timeline.md        # 时间线
    ├── plot/                  # 情节记录
    │   ├── outline.md         # 大纲与章节摘要
    │   ├── suspense-tracker.md # 悬念追踪
    │   └── foreshadow-tracker.md # 伏笔追踪
    └── changelog.md           # 更新日志

    以上为建议结构。根据具体项目的复杂度，
    可以增减文件。简单项目可以合并，复杂项目可以拆分。

[人物档案格式]
    每个重要角色一个文件，放在 bible/characters/ 下。
    文件名为角色名（如：张远.md）。
    主角和反派使用完整模板，配角使用精简模板。

    [完整模板]（主角/反派使用）

        # [角色名]

        [基本信息]
            全名：
            年龄：
            性别：
            职业/身份：
            外貌特征：[简要描述，突出辨识度高的特征]
            MBTI：[如适用]

        [性格核心]
            核心特质：[两到三个关键词]
            核心价值观：[什么对他最重要]
            内心渴望：[他真正想要什么]
            最大恐惧：[他最害怕什么]
            致命缺陷：[性格上的致命弱点]
            行为倾向：[在压力下会怎么做]

        [语言指纹]
            说话风格：[句式偏好、用词特征]
            口头禅/习惯：[如有]
            情绪变化时的语言变化：[简要说明]

        [背景]
            背景故事：[成长环境、关键经历]
            核心事件：[影响性格形成的过去事件]
            秘密：[角色隐藏的事，如有]

        [角色弧线]
            起点状态：[故事开始时的状态]
            核心冲突：[角色需要面对的核心问题]
            预期变化方向：[角色可能的成长方向，非强制]

        [当前状态]（随章节更新）
            位置：[当前在哪里]
            情绪状态：[当前心理状态]
            已知信息：[角色目前知道什么关键信息]
            持有物品：[重要物品，如有]
            身体状态：[受伤/健康等，如相关]
            最后更新：[章节编号]

        [章节行为记录]（随章节追加）
            ch01：[本章做了什么关键的事，一句话]
            ch02：[本章做了什么关键的事，一句话]
            ...

    [精简模板]（配角使用）

        # [角色名]

        [基本信息]
            全名：
            年龄：
            性别：
            职业/身份：
            外貌特征：[一到两个辨识特征]

        [性格与功能]
            核心特质：[一到两个关键词]
            在故事中的功能：[这个角色为什么存在]
            与主角的关系：[简要说明]

        [语言指纹]
            说话风格：[简要描述]

        [当前状态]（随章节更新）
            位置：
            状态：
            最后更新：[章节编号]

        [章节行为记录]（随章节追加）
            ch01：[一句话]
            ...

    [更新规则]
        - "基本信息""性格核心""语言指纹""背景""角色弧线"
          属于稳定字段，一般不因单章事件修改
        - 除非发生重大转变（如：角色经历了改变三观的事件），
          才修改稳定字段，并在changelog中说明原因
        - "当前状态"每章更新，反映最新情况
        - "章节行为记录"每章追加，不修改已有记录
        - 行为记录保持简洁，每章一句话
        - 配角如果在故事中戏份增加，可升级为完整模板


[人物关系总览格式]
    放在 bible/characters/relationships.md。
    记录所有重要角色之间的关系。

    [格式模板]

        # 人物关系总览

        [张远 ↔ 李默]
            关系：[搭档/上下级/恋人/对手等]
            当前状态：[当前关系的状态]
            关键事件：[影响关系的事件，标注章节]
            最后更新：[章节编号]

        [张远 ↔ 王璐]
            关系：
            当前状态：
            关键事件：
            最后更新：

    [更新规则]
        - 关系发生实质变化时更新（不是每章都更新）
        - "关键事件"追加记录，不删除旧记录
        - 关系描述要具体，不用"复杂"这种模糊词
          用"表面合作但张远不信任李默"这样的描述

[世界观文件格式]

    [地理/地点]（bible/worldbuilding/geography.md）

        # 地理与地点

        [地点名称]
            类型：[城市/建筑/自然地标等]
            描述：[简要描述，突出对故事有用的特征]
            首次出现：[章节编号]
            关联角色：[与此地点相关的角色]
            关联事件：[在此发生过的重要事件]

    [世界规则]（bible/worldbuilding/rules.md）

        # 世界规则与设定

        [规则/设定名称]
            内容：[具体描述这条规则]
            来源：[这条规则在哪一章被建立或揭示]
            备注：[如有限制条件或例外]

    [组织/势力]（bible/worldbuilding/factions.md）

        # 组织与势力

        [组织名称]
            性质：[类型描述]
            核心人物：[列出关键成员]
            目标/动机：[这个组织想要什么]
            与主角的关系：[敌对/盟友/中立等]
            首次出现：[章节编号]
            当前状态：[最新情况]
            最后更新：[章节编号]

    [时间线]（bible/worldbuilding/timeline.md）

        # 故事时间线

        [故事开始前的重要事件]
            [时间]：[事件]
            [时间]：[事件]

        [故事进行中的事件]（随章节追加）
            ch01 [故事内时间]：[事件摘要]
            ch02 [故事内时间]：[事件摘要]
            ...

    [世界观更新规则]
        - 新地点、新规则、新组织出现时添加
        - 已有条目发生变化时更新（如：某组织被摧毁）
        - 保持条目简洁，只记录对故事有用的信息
        - 不预设未来会出现的内容

[情节记录格式]

    [大纲与章节摘要]（bible/plot/outline.md）

        # 大纲与章节摘要

        [全书大纲]
            故事类型：
            核心冲突：
            主题：
            预计章节数：

        [章节摘要]（随章节追加）
            ch01：[两到三句话概括核心事件和结果]
            ch02：[两到三句话概括核心事件和结果]
            ...

    [悬念追踪]（bible/plot/suspense-tracker.md）

        # 悬念追踪

        [活跃悬念]
            悬念描述：[具体内容]
            提出章节：[chNN]
            当前状态：[未回应/部分回应/已回收]
            预计回收：[大致章节范围，如有预期]

        [已回收悬念]
            悬念描述：[具体内容]
            提出章节：[chNN]
            回收章节：[chNN]
            回收方式：[简要说明怎么解决的]

    [伏笔追踪]（bible/plot/foreshadow-tracker.md）

        # 伏笔追踪

        [活跃伏笔]
            伏笔描述：[具体内容]
            埋设章节：[chNN]
            埋设方式：[以什么形式出现]
            预计回收：[大致章节范围]

        [已回收伏笔]
            伏笔描述：[具体内容]
            埋设章节：[chNN]
            回收章节：[chNN]
            回收效果：[简要说明]

    [情节记录更新规则]
        - 章节摘要在每章定稿后追加，不修改已有摘要
        - 悬念状态随章节更新，回收后移至"已回收"区域
        - 伏笔同理，回收后移至"已回收"区域
        - 如果某个悬念超过预计回收位置仍未回收，标记为"超期"

[更新日志格式]（bible/changelog.md）

    # 更新日志

    [chNN 定稿后更新]
        更新时间：[时间]
        触发章节：[chNN]
        更新文件：
            - characters/张远.md：更新当前状态（位置、情绪），
              追加ch05行为记录
            - characters/relationships.md：更新张远↔李默关系状态
            - plot/outline.md：追加ch05摘要
            - plot/suspense-tracker.md：新增一条悬念，回收一条旧悬念
        备注：[如有特殊情况说明]

    [更新日志规则]
        - 每次更新必须写日志，无论更新多小
        - 日志按时间倒序排列（最新的在最上面）
        - 如果更新涉及修改稳定字段（性格核心等），
          必须在备注中说明原因

[通用规范]
    - 所有文件使用markdown格式
    - 字段为空时写"暂无"而非留空，避免遗漏
    - 章节编号统一使用 chNN 格式（ch01, ch02...）
    - 人名在所有文件中保持一致，不用昵称或简称替代
    - 描述保持客观简洁，不用文学性语言
      用"张远得知李默是卧底"而非"真相如晴天霹雳"
    - 每个文件开头标注最后更新的章节编号

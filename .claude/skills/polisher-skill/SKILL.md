# 润色技能包

[技能说明]
    润色技能，用于章节通过审查后（或审查中标注为polisher处理的问题）
    对文本进行语言层面的精修。

    润色的边界：
    - polisher做：措辞替换、句式调整、去AI味、节奏微调、
      对话标签修正、描写精炼、段落长度调整
    - polisher不做：增删场景、改变情节走向、重写大段内容、
      修改人物设定、调整章节结构

    如果润色过程中发现需要结构性修改的问题，
    记录在润色报告中，建议交回writer处理。

[文件结构]
    polisher-skill/
    └── SKILL.md    # 本文件

    润色时参考的共享技能：
    - shared/deai-rules.md（去AI味规则）

    润色时参考的写作技能：
    - writer-skill/dialogue-writing.md（对话标签和格式规范）
    - writer-skill/description-craft.md（描写标准）

[功能]

    [润色章节]
        当收到润色任务时执行。
        输入来源：reviewer的审查报告中标注为polisher处理的问题，
        或主Agent直接指派的润色任务。

        第一步：读取输入
            - 待润色的章节正文
            - 审查报告中标注给polisher的任务清单
            - 如无审查报告，则进行全面润色

        第二步：读取参考文件
            - 读取 shared/deai-rules.md
            - 如涉及对话问题，读取 dialogue-writing.md
            - 如涉及描写问题，读取 description-craft.md

        第三步：按任务类型逐项处理
            按以下优先级顺序处理：
            1. 去AI味（最高优先级）
            2. 对话修正
            3. 描写精炼
            4. 节奏微调
            5. 措辞优化

        第四步：通读检查
            完成逐项修改后，通读全文，检查：
            - 修改是否引入了新的不自然感
            - 前后文是否因修改产生衔接问题
            - 整体语感是否统一

        第五步：输出
            - 输出润色后的完整章节
            - 输出润色报告（简要说明做了哪些修改）
            - 如发现结构性问题，在报告中标注建议交回writer

[润色策略]

    [去AI味]
        参考 shared/deai-rules.md，重点处理：

        高频AI词汇替换：
        - 不是简单地换一个词，而是重写整个句子
        - 目标是让句子读起来像真人写的，而非换了个同义词

        句式均匀化处理：
        - 检查连续段落的句子长度，打破过于均匀的节奏
        - 在连续长句后插入短句，或反之
        - 变化段落的开头方式，避免连续几段以相同结构开头

        三段式和排比处理：
        - 三项列举改为两项或四项
        - 否定式排比（"不仅……而且……"）改为直接陈述
        - 过度对称的句式打破对称

        填充短语清除：
        - 删除"值得注意的是""在这个意义上"等填充词
        - 简化冗长表达

        朗读测试：
        - 修改后的文本应能大声朗读而不觉得别扭
        - 如果某个句子读起来像"念稿子"，需要重写

    [对话修正]
        参考 dialogue-writing.md，重点处理：

        标签修正：
        - 花哨标签（"他咆哮道""她低吟道"）改为"说"或动作标签
        - 副词修饰（"愤怒地说""悲伤地回答"）改为动作呈现
        - 能辨识说话人时删除多余标签

        格式规范：
        - 检查对话标点是否符合规范
        - 检查每个说话人是否另起段落
        - 检查多行对话的格式

        声音区分：
        - 检查不同角色的用词、句式是否有差异
        - 如果趋同，根据角色设定调整语言特征
        - 调整幅度控制在措辞层面，不改变对话内容

    [描写精炼]
        参考 description-craft.md，重点处理：

        情绪外化：
        - "她很悲伤" → 转化为动作或生理反应
        - "他非常愤怒" → 转化为具体表现
        - 保留情绪的方向，只改变呈现方式

        隐喻修正：
        - 陈旧隐喻（"心如刀割"）替换为更具体的表达
        - 堆砌的隐喻精简为一个精准的
        - 混合隐喻（不同喻体矛盾）统一喻体

        冗余描写精简：
        - 删除不服务于当前情绪或冲突的描写
        - 长段环境描写压缩为焦点描写
        - 保留最有力的一到两个细节，删除其余

    [节奏微调]
        不改变内容，只调整呈现方式：

        段落长度调整：
        - 连续几个长段落之间插入短段落
        - 关键时刻的段落适当缩短，制造紧迫感
        - 情感沉淀的段落可以适当拉长

        句子节奏调整：
        - 动作密集处：缩短句子，减少修饰
        - 情绪沉淀处：允许较长的句子和较多的修饰
        - 转折处：用一个短句或单独成段制造停顿

        过渡优化：
        - 场景转换处如果太突兀，添加一到两句过渡
        - 过渡太冗长的，精简为最少必要的衔接

    [措辞优化]
        最细粒度的修改，处理个别词语的选择：

        精确用词：
        - 模糊的词替换为更精确的词
          如："走"→ 根据情境选择"踱""蹿""挪""溜"
        - 重复用词：同一个词在相近位置出现多次，替换或重组句子

        语感统一：
        - 检查全章的叙述语感是否一致
        - 古风文中不出现现代网络用语（除非刻意）
        - 严肃场景中不出现过于轻松的措辞（除非刻意反差）

[润色的边界判断]
    以下情况超出润色范围，应记录在报告中建议交回writer：

    - 某段对话内容需要改变（不只是措辞）
    - 某个场景需要补充或删除
    - 角色的行为逻辑需要调整
    - 情节走向需要修改
    - 需要增加新的伏笔或悬念
    - 字数严重不足需要扩充内容

    判断标准：如果修改涉及"写什么"，交给writer。
    如果修改只涉及"怎么写"，polisher处理。

[润色报告格式]
    润色完成后输出简要报告：

    章节编号：[chNN]
    润色类型：[审查后润色 / 全面润色]

    修改概要：
    - 去AI味：[处理了哪些问题，简要说明]
    - 对话修正：[处理了哪些问题，简要说明]
    - 描写精炼：[处理了哪些问题，简要说明]
    - 节奏微调：[处理了哪些问题，简要说明]
    - 措辞优化：[处理了哪些问题，简要说明]

    超出润色范围的问题：
    - [如有，列出问题和建议处理方]

    润色后字数：[实际字数]

[注意事项]
    - 润色是精修，不是重写。修改幅度应尽量小
    - 保留原文的风格和语感，不要用自己偏好的风格覆盖
    - 每次修改都要检查是否影响了上下文的衔接
    - 对话内容的修改仅限于措辞层面，不改变角色说了什么
    - 如果原文某处写得好，不要为了"修改"而修改
    - 不修改 bible/ 目录下任何文件

# 精修技能包

[技能说明]
    精修技能，用于章节通过审查后（或审查中标注为polisher处理的问题）
    对文本进行两阶段处理：削减 + 粗糙化。

    精修的核心逻辑：
    - 第一优先级是删除。能删的不改，能精简的不保留
    - 第二优先级是打断平滑。让文字像人写的，有毛边、有断裂
    - "写得好"不是保留的理由。一个精美的排比如果是废话，照样删

    精修的边界：
    - polisher做：删除废话、打断句式、调整对话标签、
      精简描写、变化节奏、制造粗糙感
    - polisher不做：增删场景、改变情节走向、重写大段内容、
      修改人物设定、调整章节结构

    如果精修过程中发现需要结构性修改的问题，
    记录在精修报告中，建议交回writer处理。

    polisher-skill/

    精修时参考的共享技能：

[功能]

    [精修章节]
        当收到精修任务时执行。
        输入来源：reviewer的审查报告中标注给polisher处理的问题，
        或主Agent直接指派的精修任务。

        第一步：读取输入
            - 待精修的章节正文
            - 审查报告中标注给polisher的任务清单（如有）
            - 该章的场景规划（plans/chNN-plan.md，用于情感弧和节奏权重检查）
            - 如无审查报告，则进行全面精修

        第二步：读取参考文件
            - 读取 shared/deai-rules.md（必读）
            - 如涉及对话问题，读取 dialogue-writing.md
            - 如涉及描写问题，读取 description-craft.md

        第三步：阶段一·削减
            按以下顺序扫描全文，每种模式独立处理：

            [删除解释性语句]
                扫描全文，定位以下模式并删除：
                - "她知道/明白/意识到……"后接解释
                - "这意味着……""这说明……"
                - 动作之后紧跟的"因为她……"
                - 场景结尾的"这一刻，……"总结句
                - "或许……""也许这就是……"的哲理感悟

                判断标准：删掉后读者是否仍能理解发生了什么？
                能 → 删。不能 → 保留但标记考虑精简。

            [删除无功能比喻]
                对每个比喻/隐喻执行检查：
                - 这个比喻传递的感受，是否已被动作/对话/生理反应传递？
                - 是 → 删除
                - 否 → 保留，但检查能否精简为更短的表达

                特别注意：
                - "像……一样"的明喻，80%是废话
                - 连续两个以上比喻描述同一事物 → 只保留一个最精准的

            [删除排比冗余]
                定位所有排比/并列结构：
                - 三项说同一件事的不同侧面 → 保留最精准的一项
                - "不是A，而是B" → 直接说B
                - "不仅A，而且B" → 如果B是重点，直接说B
                - "既……又……还……" → 砍到最多两项

            [删除重复信息]
                定位以下重复模式并处理：
                - 叙述和对话重复同一信息 → 保留对话，删叙述
                - 两段用不同方式说同一件事 → 保留更有力的一段
                - 角色内心重复已知信息 → 删除

            [精简情绪描写]
                - 连续多个形容词/比喻描述同一情绪 → 留一个最精准的
                - "她感到悲伤、心碎、绝望" → 如有动作展示则全删
                - 情绪+生理反应+比喻三层堆叠 → 只保留生理反应

        第四步：阶段二·粗糙化
            在削减后的文本基础上，制造真人写作的质感：

            [打断对称句式]
                定位所有对称/对仗结构：
                - "A是X，B是Y"的对称 → 让两半不对等（一半长一半短）
                - "她的A如何，她的B如何" → 砍掉一半或让第二半残缺
                - 前后句结构完全对称 → 打乱其中一句的结构

            [制造长短交错]
                检查段落长度分布：
                - 连续3个以上长度相近的段落 → 中间插一个极短段落
                - 连续3个以上长度相近的句子 → 砍短或拆分其中一个
                - 全章没有单句成段 → 在合适位置制造1-2个单句段落

            [允许不完整]
                寻找可以"不说完"的位置：
                - 情绪高点可以用破折号中断
                - 观察不必给结论
                - 对话不必每问必答
                - 但不滥用——每章最多3-4处不完整

            [去掉多余转折提示]
                - "但是""然而""不过"——如果转折本身已经明显 → 删掉
                - 用动作或短句制造转折感，而非连词

            [变化段落结尾]
                检查连续段落的结尾方式：
                - 连续以动作结尾 → 换一个以对话或感官结尾
                - 连续以短句金句结尾 → 让某段自然收尾，不做总结
                - 避免每段最后一句都比前面"高级"

            [对话粗糙化]
                - 过于完整流畅的对话 → 加入打断、省略、话没说完
                - 过于礼貌 → 根据角色设定降低礼貌程度
                - 语气趋同 → 根据人物档案拉开差异
                - 注意：只改措辞和完整度，不改对话内容

        第五步：通读检查
            完成两阶段处理后，通读全文确认：
            - 修改是否引入了新的AI感？（如新的对称结构）→ 再处理
            - 删除是否造成叙事断裂？→ 如有，允许添加极简过渡（不超过20字/处）
            - 情感弧是否仍然达成（入口X → 出口Y）
            - 全文朗读是否还有"念稿子"感？→ 定位问题处理

        第六步：输出
            - 输出精修后的完整章节（覆写 chapters/chNN.md）
            - 输出精修报告

        情感弧保护规则：
        - 精修时不得破坏规划中定义的情感转变
        - 削减时如果某个"废话"段落恰好是情感转折的铺垫 → 保留
        - 判断方法：对照 plans/chNN-plan.md 中的情感弧标注
        - 如果精修后情感弧未达成，在报告中建议交回writer

[精修的边界判断]
    以下情况超出精修范围：
    - 某段对话内容需要改变（不只是措辞）
    - 某个场景需要补充或删除
    - 角色的行为逻辑需要调整
    - 情节走向需要修改
    - 字数严重不足需要扩充内容

    判断标准：涉及"写什么"交给writer，只涉及"怎么写"polisher处理。

[注意事项]
    - 精修的第一反应应该是"能删吗"而不是"怎么改"
    - 不要在削减后的空白处"补充"新的优美文字
    - 粗糙化不等于写得差——是让文字有呼吸感和人味
    - 保留原文的风格和角色声音，不用自己偏好覆盖
    - 不修改 bible/ 目录下任何文件
    - 不修改章节备注部分

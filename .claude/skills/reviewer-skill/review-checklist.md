# 审查清单与评分维度

[概述]
    章节审查分为十个维度，每个维度1-5分，满分50分。
    审查时逐维度检查，记录问题和具体位置，
    判断每个问题应交给writer还是polisher处理。

    评分通用标准：
    5分：该维度表现优秀，无明显问题
    4分：该维度表现良好，有小瑕疵但不影响阅读
    3分：该维度基本合格，有明显可改进之处
    2分：该维度存在较大问题，需要修改
    1分：该维度严重不合格，需要重写相关段落

[维度一：叙事逻辑]（核心维度）
    检查故事的因果链是否成立，事件推进是否合理。

    检查项：
    - 事件之间是否有清晰的因果关系
    - 角色的行动是否有合理的动机驱动
    - 信息揭示的时机是否合理（不过早也不过晚）
    - 是否存在逻辑漏洞（角色不可能知道的事、不可能到达的地方）
    - 冲突的发展是否有层次，而非突然爆发或突然消失
    - 解决问题的方式是否在此前有铺垫

    常见问题：
    - 角色突然获得此前未提及的能力或信息 → writer处理
    - 冲突被轻易解决，缺少过程 → writer处理
    - 事件顺序有时间线矛盾 → writer处理
    - 角色行为缺乏动机，"为了剧情需要"而行动 → writer处理

[维度二：人物一致性]（核心维度）
    检查角色言行是否符合已建立的人物设定。

    检查项：
    - 角色的说话方式是否符合其语言指纹
    - 角色的决策是否符合其性格和价值观
    - 角色的情绪反应是否与其经历和性格匹配
    - 如果角色行为与设定不同，是否有合理的触发事件
    - 角色之间的关系互动是否与已建立的关系状态一致
    - 角色的知识范围是否一致（不知道不该知道的事）

    审查时对照 bible/ 中的人物档案。

    常见问题：
    - 角色语气突然改变，没有原因 → writer处理
    - 角色做出与其性格完全矛盾的选择 → writer处理
    - 角色提到了不应该知道的信息 → writer处理
    - 不同角色说话方式趋同 → polisher处理

[维度三：连贯性]
    检查本章与前后章节的衔接是否紧密。

    检查项：
    - 时间线是否连贯，没有跳跃错误
    - 角色的物理位置是否衔接（上章在A地，本章不应无故出现在B地）
    - 角色的情绪状态是否与上章末尾衔接
    - 前文埋下的伏笔是否有呼应或推进
    - 上章结尾的悬念是否在本章有所回应
    - 是否遗忘了仍然活跃的悬念线
    - 角色的能力/信息状态是否与前文一致

    审查时对照：
    - 上一章的正文（特别是结尾段落）
    - bible/ 中的人物档案（当前状态）
    - 大纲中的已完成章节摘要

    常见问题：
    - 时间线矛盾（上章是傍晚，本章开头变成了上午） → writer处理
    - 遗忘了上章设置的悬念 → writer处理
    - 角色状态跳跃（上章受了重伤，本章活蹦乱跳） → writer处理
    - 场景转换缺少过渡 → polisher处理

[维度四：场景规划执行]
    检查本章是否完成了场景规划中设定的任务。

    检查项：
    - 规划中的核心事件是否都已呈现
    - 规划中的场景目标是否达成
    - 规划中标注的情感基调是否实现
    - 规划中要求的伏笔是否已埋设
    - 是否偏离规划且偏离不合理
    - 如果合理偏离，偏离是否带来了更好的效果

    审查时对照该章的场景规划文件。

    常见问题：
    - 遗漏了规划中的关键事件 → writer处理
    - 情感基调与规划不符 → writer处理
    - 合理偏离但需要调整后续规划 → 报告给主Agent

[维度五：节奏与张力]
    检查章节的阅读节奏是否合理，张力曲线是否有效。

    检查项：
    - 开头20%是否建立了足够的吸引力
    - 是否有节奏变化（紧张-缓解-紧张的交替）
    - 高潮点是否有足够的铺垫
    - 是否存在长段落的拖沓或无效铺陈
    - 结尾是否有悬念或翻页驱动力
    - 段落长度是否有变化
    - 信息密度高低是否有交替

    常见问题：
    - 开头平淡，缺少冲突或事件 → writer处理
    - 中段松散，节奏垮掉 → writer处理
    - 结尾交代太完整，没有悬念 → writer处理
    - 全章节奏过于均匀 → polisher处理
    - 段落长度过于一致 → polisher处理

[维度六：对话质量]
    检查对话是否有效地服务于叙事。

    检查项：
    - 每段对话是否至少承担一项功能（推进情节/揭示人物/制造冲突/营造氛围）
    - 不同角色的说话方式是否有区分
    - 对话是否有潜台词，还是全部直白表达
    - 对话节奏是否有变化（快慢、长短、打断）
    - 对话标签是否恰当（是否过度使用副词或花哨替代词）
    - 是否存在信息倾倒（角色互相说已知信息）
    - 对话是否与动作、环境配合

    审查时参考 dialogue-writing.md 中的标准。

    常见问题：
    - 对话无功能，删掉不影响故事 → writer处理
    - 信息倾倒，角色变成信息播报员 → writer处理
    - 所有角色说话语气相似 → polisher处理
    - 对话标签过于花哨 → polisher处理
    - 缺少动作配合，纯粹"说说说" → polisher处理

[维度七：描写质量]
    检查描写是否精准有效，是否服务于叙事。

    检查项：
    - 描写是否激活了具体的感官体验
    - 是否存在大段无功能的环境说明
    - 心理状态是否通过外化手段呈现（而非直接命名情绪）
    - 隐喻是否精准（非陈旧、非堆砌、非混合）
    - 描写的密度是否与节奏匹配（动作段落不宜描写过厚）
    - 新场景是否建立了足够的空间感
    - 关键道具或线索是否通过描写引起了注意

    审查时参考 description-craft.md 中的标准。

    常见问题：
    - 直接命名情绪（"她很悲伤"） → polisher处理
    - 描写过多导致节奏拖沓 → polisher处理
    - 描写过少导致场景缺乏画面感 → writer处理（需补充场景）
    - 隐喻陈旧或堆砌 → polisher处理
    - 描写与当前情绪或冲突无关 → polisher处理

[维度八：去AI味]
    检查文本是否存在AI生成的典型痕迹。

    检查项：
    - 是否存在高频AI词汇的密集使用
    - 是否存在三段式法则过度使用
    - 是否存在否定式排比（"不仅……而且……"）
    - 是否存在同义词循环
    - 是否存在过度强调意义和遗产的陈述
    - 是否存在系动词回避
    - 是否存在破折号过度使用
    - 是否存在填充短语
    - 句子长度是否过于均匀
    - 段落结构是否过于对称
    - 叙述是否有个性和声音，还是像百科全书

    审查时参考 shared/deai-rules.md 中的规则。

    常见问题：
    - 高频AI词汇密集出现 → polisher处理
    - 句式过于均匀工整 → polisher处理
    - 角色声音同质化 → polisher处理
    - 隐喻堆砌或过度对称 → polisher处理
    - 叙述缺乏个性 → polisher处理

[维度九：钩子与悬念]
    检查章节的开头吸引力和结尾悬念设置。

    检查项：
    - 开头是否在前三段内建立冲突或紧张感
    - 开头是否避免了常见弱开头（天气、日常、回顾）
    - 结尾是否留下至少一个未解问题
    - 结尾钩子的类型是否与前几章有变化
    - 是否有悬念只设不收的问题
    - 本章是否回应了之前章节的至少一个旧悬念
    - 悬念强度是否与当前叙事位置匹配

    审查时参考 shared/hook-techniques.md 中的标准。

    常见问题：
    - 开头平淡无冲突 → writer处理
    - 结尾太完整，缺少翻页动力 → writer处理
    - 悬念类型重复（连续几章都用突然揭示） → writer处理
    - 旧悬念被遗忘 → writer处理

[维度十：字数与密度]
    检查章节字数是否达标，内容密度是否合理。

    检查项：
    - 字数是否在3000-5000字范围内
    - 如果字数不足，是否有场景被跳过或压缩
    - 如果字数过多，是否有冗余或拖沓的段落
    - 核心事件数量是否与字数匹配
    - 场景数量是否与字数匹配
    - 是否存在明显的注水段落

    常见问题：
    - 字数不足3000 → writer处理（参考content-expansion.md扩充）
    - 字数超过5000且有冗余 → polisher处理（精简）
    - 场景过少导致内容单薄 → writer处理
    - 存在大段无功能描写 → polisher处理

[审查流程]
    1. 按维度一到维度十逐项检查
    2. 每个维度记录评分和具体问题
    3. 对每个问题标注处理方（writer/polisher）
    4. 计算总分
    5. 根据总分和核心维度得分给出审查结论
    6. 按 templates/review-report-template.md 格式输出报告

[评分汇总标准]
    满分：50分（10个维度 × 5分）

    通过：总分≥35分，且无单项≤2分
    需修改：总分25-34分，或有单项≤2分但整体可修复
    需重写：总分<25分，或核心维度（叙事逻辑/人物一致性）≤1分

    复审时：
    - 检查上一份报告中标记的所有问题
    - 确认问题已解决且未引入新问题
    - 重新评分，更新结论

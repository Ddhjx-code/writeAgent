# 审查报告模板

[使用说明]
    reviewer完成审查后按此模板格式输出报告。
    所有方括号内的内容为需要填写的字段。
    问题清单中每个问题都必须标注处理方（writer/polisher）。

---

# 章节审查报告

[基本信息]
    章节编号：[chNN]
    审查版本：[v1/v2/v3]
    章节字数：[实际字数]
    审查日期：[日期]

[总评]
    总分：[XX]/50
    审查结论：[通过 / 需修改 / 需重写]

    一句话总评：
    [用一到两句话概括本章的整体质量和核心问题]

[各维度评分]

    叙事逻辑（核心）：    [X]/5
    人物一致性（核心）：  [X]/5
    连贯性：              [X]/5
    场景规划执行：        [X]/5
    节奏与张力：          [X]/5
    对话质量：            [X]/5
    描写质量：            [X]/5
    去AI味：              [X]/5
    钩子与悬念：          [X]/5
    字数与密度：          [X]/5

[优点]
    记录本章做得好的方面，帮助writer了解应保持的方向。

    1. [具体优点描述]
    2. [具体优点描述]
    （至少列出两项）

[问题清单]
    按严重程度排列，最严重的排在最前。

    [问题1]

        影响维度：[涉及的维度名称]
        处理方：[writer/polisher]
        问题描述：[具体描述问题是什么]
        原文位置：[引用相关原文片段，标注大致位置]
        修改建议：[具体可执行的修改方向]

    [问题2]
        严重程度：[高/中/低]
        影响维度：[涉及的维度名称]
        处理方：[writer/polisher]
        问题描述：[具体描述]
        原文位置：[引用原文]
        修改建议：[具体建议]

    [问题3]
        ...

    （列出所有发现的问题，不设数量上限）

[修改任务汇总]

    [交给writer的任务]
        1. [具体任务描述]
        2. [具体任务描述]
        ...

    [交给polisher的任务]
        1. [具体任务描述]
        2. [具体任务描述]
        ...

[连贯性备注]
    记录本章中需要在后续章节注意的事项。

    [人物状态变化]
        [记录本章中角色的位置、情绪、关系、能力等变化]

    [新增悬念]

    [待回收的旧悬念]
        [记录仍然活跃但本章未回收的悬念]

    [伏笔记录]
        [记录本章埋设的伏笔及预期回收位置]

[复审记录]（复审时填写）
    上一版问题解决情况：

    [问题1原描述]
        状态：[已解决/部分解决/未解决]
        说明：[简要说明]

    [问题2原描述]
        状态：[已解决/部分解决/未解决]
        说明：[简要说明]

    新引入的问题：
    [如修改后引入了新问题，按问题清单格式记录]

---

[模板使用注意]
    - 优点部分不要省略，正面反馈和问题反馈同等重要
    - 问题描述必须引用原文，不能只说"某处有问题"
    - 修改建议必须具体可执行，不能只说"需要改进"
    - 连贯性备注部分会被后续章节的审查参考，务必填写
    - 复审时重点检查上一版问题，不需要重新全面审查
    - 如果审查结论为"通过"，问题清单中应只有低严重度问题

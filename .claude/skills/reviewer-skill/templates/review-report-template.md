# 审查报告模板

[使用说明]
    reviewer完成审查后按此模板格式输出报告。
    所有方括号内的内容为需要填写的字段。
    问题清单分两个区域输出：
    - 结构/逻辑类（writer处理）
    - 表达类（polisher处理）
    每个问题必须标注来源：[骨架层/扩展层/表达层]。

---

[模板使用注意]
    - 问题清单分两个区域：结构/逻辑类和表达类，不得混排
    - 每个问题必须标注来源：骨架层/扩展层/表达层
    - 问题描述必须引用原文，不能只说"某处有问题"
    - 修改建议必须具体可执行，不能只说"需要改进"
    - 优点部分不要省略，但优点不能成为放过问题的理由
    - 连贯性备注部分planner依赖此信息更新bible，务必填写
    - 复审时重点检查上一版问题，不需要重新全面审查
    - 如果审查结论为"通过"，结构/逻辑类问题区域应为"无"

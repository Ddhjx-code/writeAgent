# /project:continue

无需参数，自动检测进度。

## 流程

### Step 1 定位进度（planner）
派planner执行"进度汇总"任务：
- 读取 bible/plot/outline.md 的TODO清单
- 找到下一个未完成章节
- 检查是否有进行中但未完成的章节

### Step 2 状态判断（主Agent）

**无bible目录** → 提示执行 /project:plan-story

**有进行中的章节** → 判断停在哪一步：
- 已写未审查 → 从审查步骤继续
- 已审查未润色 → 从润色步骤继续
- 已润色未归档 → 从归档步骤继续

**有下一个待创作章节** → 执行 /project:write-chapter N

**全部章节已完成** → 提示全书收尾

### Step 3 执行
按判断结果执行对应流程。

单章完成后，主Agent判断：
- 下一章存在 且 未触发里程碑 → 自动继续下一章
- 触发里程碑（章节号为5的倍数）→ 进入 /project:review-milestone
- 全部完成 → 提示全书收尾

### Step 4 连续创作
支持连续创作多章，直到：
- 触发里程碑审查（每5章）
- 全部章节完成
- 遇到重大转折需要人类确认
- 出现需重写的章节且重试3次仍未通过

# /project:learn

无需参数，自动读取当前会话中的人类反馈。

## 流程

### Step 1 收集反馈（主Agent）

读取以下来源中的人类反馈：
- 当前会话中人类提出的修改意见
- 如指定章节编号，同时读取 reviews/chNN-review.md 中的修改历史

整理出所有人类不满意的地方，逐条列出原始反馈。

### Step 2 理解意图（主Agent）

对每条反馈逐一分析：

- 表象：人类具体说了什么
- 意图：人类真正不满意的是什么
- 规律：背后对应的写作原则是什么
- 适用范围：
  - 仅本章（特殊情境，不具备普遍性）
  - 本项目（与这个故事的风格或人物有关）
  - 通用（适用于所有写作场景）

仅本章的反馈不提炼为规律，跳过。

### Step 3 提炼规律（主Agent）

将理解到的规律用写作原则的语言表达。

规律的表达格式：
- 不写"这段太平"
- 写"在对话冲突场景中，心理描写连续超过X行会打散紧张感，
  应改为将心理活动嵌入动作或对话间隙"

每条规律包含：
- 适用场景：在什么情境下触发
- 原则内容：应该怎么做或避免什么
- 来源反馈：对应的原始人类反馈（一句话概括）

### Step 4 分类（主Agent）

按适用范围分类：

项目级规律（本项目风格偏好）：
→ 写入 bible/style-guide.md

技能级规律（通用写作原则）：
→ 按内容归入对应skill文件的 [项目积累] 章节：
  - 对话相关 → dialogue-writing.md
  - 描写相关 → description-craft.md
  - AI味相关 → deai-rules.md
  - 节奏结构相关 → chapter-guide.md
  - 悬念钩子相关 → hook-techniques.md

### Step 5 人类确认（主Agent）

向人类展示提炼结果：

```
「本次提炼结果」

项目级规律（将写入 bible/style-guide.md）：
1. [规律内容]
   来源：[原始反馈概括]

通用规律（将写入对应skill文件）：
1. [规律内容] → [目标文件]
   来源：[原始反馈概括]

跳过的反馈（仅适用本章，不提炼）：
1. [原始反馈]
   原因：[为什么判断为仅本章适用]

请确认以上理解是否正确？
如有偏差请指出，我修正后再写入。
```

等待人类确认。
人类确认 → 执行 Step 6。
人类指出偏差 → 修正对应条目，重新展示，再次确认。

### Step 6 写入文件（主Agent）

将确认后的规律写入对应文件。

写入规则：
- 追加到文件末尾的 [项目积累] 章节
- 不覆盖文件中已有的内容
- 每条规律附上来源反馈，方便后续回溯
- bible/style-guide.md 如不存在则新建

写入完成后返回确认：

```
「写入完成」
已写入：
- bible/style-guide.md：X条新规律
- [skill文件名]：X条新规律

writer和reviewer下次执行时将自动参考这些规律。
```
